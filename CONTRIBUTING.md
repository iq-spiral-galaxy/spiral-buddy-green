# 기여 가이드 (Contributing)

Spiral Buddy Green에 관심 가져주셔서 감사합니다. PR·이슈 모두 환영합니다.

## ⚖️ 기여 전 꼭 읽어주세요 — 라이선스 동의

이 프로젝트에 기여물을 제출하면 **[기여자 라이선스 계약(CLA)](CLA.md)** 과 아래
**개발자 원산지 증명(DCO)** 에 동의한 것으로 간주됩니다. 요약하면:

- 기여물의 저작권은 **기여자 본인이 그대로 보유**합니다.
- 다만 프로젝트 소유자(및 그 승계인·양수인)에게 기여물을 **자유롭게 이용·재라이선스(상업용·
  독점 라이선스 포함)** 할 수 있는 폭넓은 권리를 부여합니다.
- 코드 기여는 회사 **지분이나 금전 보상과 무관**합니다.

전체 조건은 [CLA.md](CLA.md)에 있습니다. 동의할 수 없다면 PR을 제출하지 말아 주세요.

## 🚀 기여 절차

1. 이슈로 먼저 논의 (큰 변경일수록 권장).
2. 포크 후 feature 브랜치 생성: `git checkout -b feat/내-기능`
3. 변경 후 **반드시 sign-off 를 포함해 커밋**:
   ```bash
   git commit -s -m "feat: 내 기능 설명"
   ```
   `-s` 플래그가 커밋 메시지 끝에 `Signed-off-by: 이름 <이메일>` 라인을 자동으로 붙입니다.
   이 라인이 DCO 동의의 증거이며, CI(DCO 검사)가 이를 확인합니다.
4. `pnpm install && pnpm build` 로 빌드가 통과하는지 확인.
5. PR을 열고 템플릿의 동의 항목을 체크.

> 💡 sign-off를 깜빡했다면 `git commit --amend -s` (마지막 커밋) 또는
> `git rebase --signoff main` (여러 커밋)으로 추가할 수 있습니다.

## <a name="dco"></a>📜 개발자 원산지 증명 (Developer Certificate of Origin 1.1)

커밋에 `Signed-off-by:` 를 추가함으로써, 기여자는 아래 DCO에 동의함을 증명합니다:

```
Developer Certificate of Origin
Version 1.1

기여자는 sign-off 를 통해 다음을 증명합니다:

(a) 본인이 해당 기여물을 작성했으며, 프로젝트의 오픈소스 라이선스로 제출할 권리가 있다;
    또는
(b) 본인이 아는 한, 해당 기여물은 적절한 오픈소스 라이선스로 보호되며 본인이 그 라이선스
    하에 (본인이 작성했든 변경했든) 제출할 권리가 있다; 또는
(c) 본 기여물은 (a) 또는 (b)를 증명한 다른 사람으로부터 직접 제공받았으며, 본인은 이를
    변경하지 않았다.

(d) 본 기여와 sign-off 가 공개되며, 기여물에 첨부된 기록(sign-off 포함)이 무기한
    유지되고 프로젝트의 라이선스에 따라 재배포될 수 있음을 이해하고 동의한다.
```

(원문: https://developercertificate.org/)

## 🏷️ 코드 스타일

- 기존 코드의 주석 밀도·네이밍·관용구를 따라주세요.
- 커밋 메시지: `feat:` / `fix:` / `docs:` / `refactor:` 등 prefix 사용.
- 사용자 향 텍스트는 한국어, 코드 주석도 한국어 위주(기존 코드 참고).
